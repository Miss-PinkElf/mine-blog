# 复盘型知识文档（Review Knowledge Doc）

## 使用场景

用于把一段对话整理成可回看的知识文档。重点不是把聊天逐字复述，而是沉淀背景、决策、例子、取舍和后续行动。

## 推荐结构

```markdown
# 主题-复盘型知识文档-日期

## 背景

## 前因后果

## 关键决策

## 为什么这么做

## 为什么不那么做

## 例子说明

## 知识延伸

## 后续行动
```

## 写作要求

- 背景要说明这件事为什么出现，而不是只写“用户要求总结”。
- 前因后果要保留决策链路：先发生什么、后来为什么改变、最终如何收敛。
- 关键决策要写成清晰条目，每条包含决策、原因和影响。
- 例子说明（examples）要贴近当前对话，避免泛泛举例。
- 取舍分析（trade-off analysis）必须解释“为什么这么做”和“为什么不那么做”。
- 知识延伸（knowledge extension）控制在 1-3 个点，每个点说明和当前主题的关系。

## 大纲确认要求

写文件前先给用户看大纲。大纲至少包含：

- 拟定文件名
- 文档主线
- 关键决策列表
- 计划延伸的 1-3 个知识点
