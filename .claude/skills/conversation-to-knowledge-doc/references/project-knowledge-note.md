# 项目沉淀文档（Project Knowledge Note）

## 使用场景

用于把工程讨论、方案收敛、任务推进或流程规范沉淀成项目内可维护的知识文档。重点是帮助后来的人理解背景、约束、决策和后续影响。

## 推荐结构

```markdown
# 主题-项目沉淀文档-日期

## 背景

## 目标

## 约束

## 方案

## 决策记录

## 涉及文件或模块

## 取舍分析

## 例子说明

## 风险与后续事项

## 知识延伸
```

## 写作要求

- 背景要说明任务为什么存在，以及它和项目当前状态的关系。
- 约束要列出明确影响方案的限制，例如流程门禁、路径约束、验证要求。
- 决策记录要包含：决策内容、决策原因、影响范围。
- 涉及文件或模块只列用户明确引用或对话中已经确认的内容；不要主动猜测。
- 取舍分析（trade-off analysis）要解释已选方案和未选方案的差异。
- 例子说明（examples）要服务维护者理解，例如给出文件命名、流程执行或输出结构示例。
- 知识延伸（knowledge extension）要偏项目实践、流程设计、技能设计或工程维护。

## 大纲确认要求

写文件前先给用户看大纲。大纲至少包含：

- 拟定文件名
- 项目背景摘要
- 决策记录列表
- 涉及文件或模块
- 风险与后续事项
