# Learnings · oc-mugen-rin-character-sheets

1. **设定图优先于单次好看立绘**：cat-oc-design 的核心是用「全局/角度表情/动作/场景」约束后续生成一致性；每次生成应带回原始设定图，避免 A→B→C 漂移。
2. **模型约束要听用户环境**：本环境用户确认只有 `gpt-image-2` 可用；不要擅自换 host 模型「碰运气」。
3. **中转 500 的判断顺序**：先看 `/models` 是否 200 → 再试极简 images/responses → 换 key → 清代理/noproxy → 判上游挂掉。本轮在「换 key + noproxy」后仍 500，即可认定外部阻塞。
4. **图生图体积**：参考图先压到可接受体积再 `--image`；skill 请求体用临时文件 `@file`，禁止把大 base64 塞进 shell 变量。
5. **角色视觉锚点（无限凛）**：
   - 黑发 + 荧光绿/赛博绿发丝
   - 异色瞳：一黑一绿；绿瞳 ∞ 符号
   - 色板：黑 / 赛博绿 / 深灰 JK
   - 服装：魔改 JK；宅家黑卫衣 + 一黑一绿过膝袜 + 耳机
   - 气质：三无/面无表情为主，偶发傻萌/傲娇
