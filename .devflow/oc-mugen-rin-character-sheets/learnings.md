# Learnings · oc-mugen-rin-character-sheets

1. **设定图优先于单次好看立绘**：cat-oc-design 的核心是用「全局/角度表情/动作/场景」约束后续生成一致性；每次生成应带回原始设定图，避免 A→B→C 漂移。
2. **模型约束要听用户环境**：本环境用户确认只有 `gpt-image-2` 可用；不要擅自换 host 模型「碰运气」。
3. **中转 500 的判断顺序**：先看 `/models` 是否 200 → 再试极简 images/responses → 换 key → 清代理/noproxy → 判上游挂掉。本轮在「换 key + noproxy」后仍 500，即可认定外部阻塞。
4. **图生图体积**：参考图先压到可接受体积再 `--image`；skill 请求体用临时文件 `@file`，禁止把大 base64 塞进 shell 变量。
5. **角色视觉锚点（无限凛）**：
   - 黑发 + 荧光绿/赛博绿发丝
   - 异色瞳：一黑一绿；绿瞳 ∞ 符号
   - 色板：黑 / 赛博绿 / 深灰 JK
   - 服装：魔改 JK；宅家黑卫衣 + 一黑一绿过膝袜 + 耳机
   - 气质：三无/面无表情为主，偶发傻萌/傲娇

6. **HTTP 524 ≠ 本地等太短**：日志与 `failed-*.json` 会出现 `error code: 524`；`CURL_MAX_TIME` 往往够长，是网关先断。
7. **复杂 multi-panel 更易 524**；单场景（JK 立绘、动作 4 格）常 1 次成功；表情大 sheet 可能要多次重试。
8. **pgrep 等待脚本陷阱**：wait 循环命令行若含 `run.sh` 字符串，会误匹配自身导致假死循环；应用更精确的进程匹配。
9. **prompt-2 出图策略（2026-07-17）**：
   - 风格参考先整包下载 `ref-style/`，再谈画风对齐
   - 大设定图做 `--image` 请求体 ~1MB+ 时 524 率高；日常切片用 **text 模式**更稳
   - 用户要求「单次最多 300s、不要长等」→ `--retries 0`；失败立刻简化构图再试，不要默认 5 次重试
   - 文件名可保留批次意图（如 live-stream），若实际简化题材须在 state/handoff 写明
10. **质量预期**：第一版日常/表情包可先落盘；观感不满意时记「暂接受第一版、后续重抽」，不要在中转不稳时无限磨一张
