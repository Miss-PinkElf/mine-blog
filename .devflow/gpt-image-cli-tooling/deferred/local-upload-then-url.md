# 延期 · 本地大图先上传再 URL

## 状态

明确延期（2026-07-18 官方文档 Explore 后记录）

## 现象 / 动机

- 官方图生图示范使用 **https `image_url`**，body 极小
- skill 当前用 **data URL base64**，本地大图/多图易触发网关断连
- 若做「先上传到可公网读的 URL 再引用」，可显著稳多图/大图

## 为何延期

- 需要上传存储/鉴权/生命周期策略（中转是否提供、是否自建）
- T7 第一版优先 `metadata` size/quality；`--prep` 已能缓解部分 body 问题
- 与「内嵌运行时 / 输出体积治理」同属更大基础设施

## 当前替代

- `--prep heavy` / 调小 target / 重试
- 若参考图已是公网 URL：下轮可选 CLI 直通（不 base64）

## 何时重开

用户需要稳定多张大参考图、且 prep 仍不够时再立项。
